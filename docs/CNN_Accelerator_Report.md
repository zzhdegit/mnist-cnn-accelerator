# MNIST CNN 硬件加速器项目报告

## 1. 项目概述
本项目实现了一个基于 FPGA 的卷积神经网络（CNN）加速器，专门用于识别 28x28 像素的手写数字图片。该加速器完全由 SystemVerilog 编写，并经过了比特对齐（Bit-accurate）的端到端仿真验证。

## 2. 硬件架构设计
加速器采用流水线流式处理架构，包含以下核心模块：
- **Conv1 Layer**: 1输入通道，32输出通道，3x3卷积核。采用 Shift-Register Line Buffer 结构。
- **Conv2 Layer**: 32输入通道，64输出通道，3x3卷积核。支持高并行 32 通道输入累加。
- **MaxPool Layer**: 2x2 窗口，步长为 2，实现特征降维。
- **FC1 Layer**: 全连接层，9216 输入节点，128 输出节点。采用内部 ROM 存储权重。
- **FC2 Layer**: 全连接层，128 输入节点，10 输出节点（最终分类得分）。

## 3. 量化与精度
- **量化策略**: Q8.8 定点化（16 位有符号整数，低 8 位为小数部分）。
- **测试结果**:
    - **测试样本量**: 1000 张 MNIST 原始测试图片。
    - **硬件正确识别数**: 983 张。
    - **最终准确率 (Accuracy)**: **98.30%**。

## 4. 运行性能说明
- **仿真耗时原因**: 在 PC 上运行 Vivado 仿真非常缓慢，是因为通用 CPU 需要用软件模拟 RTL 的每一个信号边沿、每一个门电路的翻转。这与 RTL 代码本身的性能无关。
- **硬件预期性能**: 一旦部署到 FPGA 硬件上，该 RTL 代码将利用硬件并行性，在 100MHz 时钟下，识别一张图片仅需数万个时钟周期（约 **0.5 毫秒** 左右），比仿真快数万倍。

## 5. 存储位置
- **源代码**: `fpga/src/`
- **仿真文件**: `fpga/sim/`
- **量化数据**: `fpga/data/`
- **一键仿真脚本**: `fpga/scripts/sim.tcl`
